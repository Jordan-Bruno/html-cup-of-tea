HTML Structure

Every HTML document starts with <!DOCTYPE html> to tell the browser it's reading an HTML5 file. The whole page sits inside <html>. Inside that, we split the content into a <head> section for meta data and a <body> section for visible content.

The <head> can include the title, stylesheets, fonts, or meta tags. The <body> is where all headings, text, images, and other visible parts go.

Headings and Text

HTML provides six levels of headings from <h1> (most important) to <h6> (least important). Use headings to give structure and meaning to your page. For regular text, use the <p> tag.

Try to use headings in order. Start with one <h1> per page, then use <h2> and lower as needed for subtopics.

Line Breaks and Emphasis

To break lines inside a paragraph, use <br />. Use <em> to italicise text (used for emphasis), and <strong> to bold it (used for importance).

These are semantic tags - they also help screen readers know which parts of the text are emphasised or important.

Using <div> and Semantic Elements

<div> is a generic container often used to group content. However, HTML also includes semantic tags like <section>, <article>, and <main> that help explain the meaning of the content.

Try to use semantic tags where possible. They make your code easier to read and better for accessibility and search engines.

Lists: Ordered and Unordered

Use <ul> (unordered list) for bullet points, and <ol> (ordered list) for steps or rankings. Each item in the list must go inside an <li> (list item) tag.

Lists help organise content clearly. Always ensure the structure is correct: the list goes in a <ul> or <ol> and each entry inside is an <li>.

Images and Attributes

Use <img> to display images. Images can be loaded from the internet using a full URL, or locally using a file path. Use the alt attribute to describe the image - this is important for accessibility.

Other common image attributes include width, height, and title. It is best practice to resize images using CSS to keep control over layout and design without distorting the image.

Videos and More Attributes

The <video> tag lets you add a video player. Include the controls attribute so users can play, pause, and adjust the video. Like images, videos can also use local file paths or external URLs.

Always include fallback text between the <video> tags for users whose browsers don't support HTML5 video.
